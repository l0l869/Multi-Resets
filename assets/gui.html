General Info


Supported auto-reset versions: 1.19.50, 1.16.10, 1.16.1, 1.16.0.58, 1.16.0.57, 1.16.0.51, 1.14.60, 1.2.13


Reset Methods

  • Setup

    • Pixel coordinates are set with their respected colour to determine the current screen and click.


  • Setupless

    • Finds English text, determining the buttons to press. Make sure MC GUI scaling is at the highest value.


  • Reading Screen Memory

    • Reads the game memory to get the current screen, relies on setup click data for the clicks. Buggy and should only be used if nothing else works. Supported: 1.16.1, 1.2.13



Manual Resetting

  • To enter an instance, press the stop reset hotkey while hovering over your desired instance
  • To reset all instances, press the reset hotkey while on the wall screen

Cumulative Resetting

  • Automatically resets and queues up more instances until you hit "stop reset," allowing you to cycle through the queued instances. This means you can go off on a break and come back with more instances ready at hand.
  • Disable auto-restart and coop mode. It's also recommended to have some switch delay to prevent instances crashing.

Affinity

  • Set the threads utilisation to the (%) amount of CPU threads utilised during resets
  • All CPU threads will be utilised during runs
  • For efficient resetting, the recommended number of instances: Total CPU Threads / 4

Timer

  • Colours are inputted in the hexadecimal ARGB format
  • Animation Length can be omitted, if you don't want the timer to be animated
  • Auto Splitting works for 1.16.1 and 1.16.10
